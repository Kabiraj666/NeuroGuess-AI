Project Overview

NeuroGuess AI is an interactive human-in-the-loop prediction simulator designed to calibrate human decision boundaries against randomized mathematical systems. Styled as a futuristic operating system dashboard, the project represents a practical experiment in Human-AI Interaction, telemetry pattern modeling, and behavioral analytics mapping.

The system records real-time input offset deviations, calculates binary bounds reduction ratios, and visualizes individual decision paths using responsive web widgets.

AI/ML Engineering Concepts

This B.Tech CSE (AI & ML) major project implements a real-world predictive ML pipeline combined with client-server data architecture:

  • Heuristic Binary Partitioning: The active drill relies on a binary search bounds minimization tree. Every input filters search space entropy by half at peak efficiency:
    Optimal Attempts = ⌈log₂(Range Max - Range Min + 1)⌉
  • Supervised Machine Learning Classifier: Predicts the probability of winning the current calibration run based on real-time features. We trained a Logistic Regression Model (scikit-learn) with balanced class weights using simulated gameplay decision paths.
  • Feature Engineering Pipeline: The model processes 5 custom features at inference:
    1. Normalized Distance = |Guess - Target| / InitialRange
    2. Attempts Remaining Ratio = TriesLeft / MaxTries
    3. Sequence Index = Current Guess Number
    4. Boundary Entropy Reduction = CurrentRange / InitialRange
    5. Direction Consistency (Verification of player adhering to feedback)
  • Client-Server Telemetry Database: All gameplay decision states are logged via an HTTP POST request to a Flask backend, saving records to a SQLite database. This creates a data-collection loop that expands the training dataset dynamically over sessions.
  • Explainable AI (XAI) & Print Audits: Outputs live calculation logs to demystify mathematical boundary adjustments in real-time, matching modern regulatory principles of explainability.
